# TORU — PDF共有確認画面 仕様書

## 概要

プロジェクト/現場グループへのコンテンツ共有時に表示する確認モーダルの仕様。
**一律禁止ではなく、ユーザー判断で安全に共有できるようにする** のが目的。

背景:
- グループには同業他社が混在する可能性がある
- 単価/見積書情報の漏洩は仕事を失う直接リスク
- 一方で、信頼できるグループでは見積書の共有も実用上必要
- → ユーザーに状況判断させる確認画面で両立する

作成: 2026-04-19

---

## 設計・方針

### 3段階の確認強度

| レベル | アイコン | 対象コンテンツ | 確認強度 |
|--------|---------|--------------|---------|
| GREEN | 🟢 | 図面PDF / 施工メモ | 軽め |
| YELLOW | 🟡 | 拾い出し結果（金額抜き） | 中 |
| RED | 🔴 | 見積書 / 単価入りPDF / 金額付き拾い出し | 高（赤アラート） |

### 判定ロジック

```
1. コンテンツの content_type を確認
   - drawing       → GREEN候補
   - memo          → GREEN候補
   - extraction    → contains_price で分岐
                     - false → YELLOW
                     - true  → RED
   - quote         → 必ずRED

2. 自動検出（追加スキャン）
   - PDF本文・OCRテキストに以下キーワードを検索（下記「自動検出キーワード」参照）
   - 1つでもヒット → 強度を RED に昇格（GREEN/YELLOW判定を上書き）

3. 信頼グループ補正（trust_level による緩和）
   - project_groups.trust_level = 'trusted' の場合:
     - GREEN/YELLOW はそのまま
     - RED → YELLOW に緩和（ただしメンバー一覧表示は維持）
     - 小さな現場・身内グループでの使用感を軽くする
   - trust_level = 'mixed' の場合:
     - 通常判定どおり（3段階フル適用）

4. 業種タグ補正（同業他社判定）
   - 共有者の company/user の industry_tag と、グループメンバー各社の industry_tag を比較
   - 同じ業種が2社以上存在 → RED モーダルで「同業他社の可能性」を明示表示
   - 異なる業種のみ → 「異業種のみ」と表示（注意喚起やや弱）

5. 確定した強度でモーダル表示
```

### 安全装置

1. **メンバー可視化**: 確認画面に「今このグループにいる会社一覧」を必ず表示
2. **共有ログ**: `share_logs` テーブルに判断時のスナップショット記録（誰が・いつ・何を・どの強度で・どのメンバーがいた状態で）
3. **共有取り消し**: 共有後も `revoked_at` で取り消し可能（メンバーは閲覧不可になる）
4. **自動検出のbypass禁止**: 検出ロジックはサーバー側で実行、クライアント改変では回避不可

### モーダル仕様

#### 🟢 GREEN モーダル

```
┌─────────────────────────────────────┐
│ このグループに共有しますか？         │
│                                     │
│ ファイル: drawing-001.pdf           │

```

```
│ 拾い出し結果を共有します             │
│                                     │
│ 種類: 材料・数量リスト（金額なし）    │
│                                     │
│ ▼ 共有内容プレビュー                │
│   - 塩ビ管 VU50: 12m                │

│   - ...                             │
│                                     │

│ メンバー: A工務店, B設備, C電気      │
│                                     │
│   [キャンセル]      [共有する]       │
└─────────────────────────────────────┘
```

#### 🔴 RED モーダル

```
┌──────────────────────────────────────────┐
│ ⚠️ 金額情報が含まれます                  │
│                                          │
│ ファイル: quote-2026-04-19.pdf           │
│ 種類: 見積書                             │
│ 検出: 「¥」「単価」キーワードを検出      │
│                                          │
│ ───────────────────────────────────      │
│ 🚨 このコンテンツは以下の会社全員が       │
│    閲覧できるようになります：            │
│                                          │
│   ・A工務店（御社）         [配管]       │
│   ・B設備                   [電気]       │
│   ・C電気（同業他社の可能性）[配管] ⚠    │
│                                          │
│ 単価情報の漏洩は競争上のリスクに         │
│ つながる可能性があります。               │
│ ───────────────────────────────────      │
│                                          │
│ ☐ 上記を確認しました                    │
│                                          │
│   [キャンセル]      [理解した上で共有]   │
└──────────────────────────────────────────┘
```

- チェックボックスを入れないと「共有」ボタンが押せない（誤操作防止）
- 「共有」ボタンは赤色
- 各メンバー名の横に `[業種タグ]` を表示、共有者と同業種には `⚠` アイコン

#### trusted グループでの緩和モーダル（RED → YELLOW風）

```
┌──────────────────────────────────────────┐
│ 💼 金額情報を共有します                   │
│                                          │
│ このグループは「信頼グループ」です。     │
│ メンバー: A工務店, 兄弟会社B, 協力店C    │
│                                          │
│ ▼ 共有内容プレビュー                     │
│   [見積書プレビュー]                     │
│                                          │
│   [キャンセル]      [共有する]           │
└──────────────────────────────────────────┘
```

- trusted フラグが付いている場合はチェックボックスなし、ボタン色も通常色

### 自動検出キーワード（初期セット）

| 言語 | キーワード |
|------|-----------|
| 日本語 | ¥, 円, 単価, 見積, 金額, 価格, 合計, 小計, **税込**, 税抜, 割引, 値引, 請求, 報酬, 原価 |
| 英語 | total, amount, price, subtotal, USD, JPY, invoice, quote |
| 数値パターン | `¥[0-9,]+`、`[0-9,]+円`、`¥[0-9,]+(\\.[0-9]+)?` |

検出は OR 条件、1つでもヒットで RED 昇格。
※ 運用しながらキーワード追加・調整可能（`keyword_config` テーブルでリモート管理も検討）

### 共有ログのデータ構造

```jsonb
{
  "confirmation_level": "red",
  "auto_detected_keywords": ["¥", "単価"],
  "member_snapshot": [
    {"company_id": "...", "company_name": "A工務店"},
    {"company_id": "...", "company_name": "B設備"},
    {"company_id": "...", "company_name": "C電気"}
  ],
  "action": "shared",
  "user_acknowledged": true
}
```

### 共有取り消しフロー

1. 共有者が「共有取り消し」を実行
2. `group_shares.revoked_at` に現在時刻を記録
3. グループ画面から該当アイテムが消える
4. 既に閲覧された履歴は残る（完全な「なかったこと」にはできないが、それも明示する）
5. `share_logs` に `action: 'cancelled'` で追記

## UI実装ノート（React Native / Expo）

### コンポーネント構成（案）

```
<ShareConfirmationModal>
  ├─ <ContentSummary />     // ファイル名・種類・プレビュー
  ├─ <MemberList />         // 共有先メンバー会社一覧
  ├─ <RiskWarning />        // RED時のみ表示
  ├─ <AcknowledgeCheckbox /> // RED時のみ表示
  └─ <ActionButtons />      // キャンセル/共有
</ShareConfirmationModal>
```

### 状態管理

- 確認強度: `'green' | 'yellow' | 'red'`
- 自動検出キーワード: `string[]`
- 確認チェック: `boolean`（RED時の有効化条件）
- メンバースナップショット: グループ参照時に取得
